# Design Decision Records (DDRs)

This is the decision log for UI/UX design choices. DDRs are scoped to design
trade-offs (layout, interaction patterns, visual language); architectural and
implementation trade-offs live in [`../adr/`](../adr/README.md) instead.

Status values: **Accepted**, **Proposed**, **Deprecated**, **Superseded**.
When a record is superseded, both the old and new entries remain in the log —
the old one keeps its number and is marked accordingly.

New DDRs follow the structure shown by existing records in this folder and use
the next available number.
